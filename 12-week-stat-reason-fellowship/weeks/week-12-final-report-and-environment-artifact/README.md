# Week 12: Final report and demo

## Goal

Turn the work into a public-quality artifact: clear question, method, results, failures, and next work.

## Required watching

- Revisit one talk from the course that changed how you think. Write why.

## Required reading

  - ../../prime-intellect-framework.md

  - ../../project/final-report-template.md
- Current field notes.
  - ../../current-field-notes.md

## Project exercises

Create `project/reports/final-report.md`.

1. Use `project/final-report-template.md`.
2. Include task suite, Prime environment structure, verifier design, baseline results, error analysis, improvement loop, limitations, and next work.
3. Add at least 3 concrete failure examples from week 10 or 11.
4. Add a verifier reliability section that names known blind spots.
5. Add a limitations section that names the weakest parts of the work.

Package the artifact.

1. Write the exact Prime eval command or demo steps needed to reproduce the final result.
2. Check task IDs, run IDs, reported metrics, Prime artifacts, and source links.
3. Confirm that the environment README explains what the task suite measures.
4. Confirm that a reader can find the raw baseline outputs and final reported metrics.
5. Decide whether the environment should be published, kept private, or cleaned up before publishing.

## Deliverable

- Final report.
- Prime environment demo or reproducibility instructions.
- Clean task suite and baseline results.

## Mastery check

You should be able to explain the project in 5 minutes, run or point to the Prime eval artifact, then defend the measurement choices under pushback.
