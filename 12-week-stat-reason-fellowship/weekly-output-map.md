# Weekly output map

Use this file as the running checklist for the capstone. Each week also produces `project/reports/week-XX-three-author-notes.md`, a short synthesis from the three-author loop.

| Week | Project stage | Required artifacts | Done when |
| --- | --- | --- | --- |
| 01 | Task ideation from probability | `project/notebooks/week-01-probability-simulation.*`, `project/tasks/week-01-task-ideas.md`, `project/reports/week-01-three-author-notes.md` | 3 simulations match exact answers, 5 task stubs have verifier ideas, and probability is synthesized from 3 authors |
| 02 | Distribution task seeds | `project/notebooks/week-02-distributions.*`, `project/tasks/week-02-distribution-tasks.md`, `project/reports/week-02-three-author-notes.md` | 2 tasks have reference answers, verifier logic, malformed-answer examples, and a 3-author distribution summary |
| 03 | Inference task seeds | `project/tasks/week-03-inference-tasks.md`, `project/reports/week-03-inference-glossary.md`, `project/reports/week-03-three-author-notes.md` | 3 tasks cover CI, p-value, and sample size or power, with inference concepts triangulated across 3 authors |
| 04 | ML evaluation tasks | `project/notebooks/week-04-model-evaluation.*`, `project/tasks/week-04-ml-eval-tasks.md`, `project/reports/week-04-three-author-notes.md` | 3 model-eval tasks include misleading metric or leakage cases, and the metric vocabulary is reconciled across 3 authors |
| 05 | Prime setup and model lifecycle | `project/reports/week-05-model-lifecycle.md`, `project/reports/week-05-prime-setup.md`, `project/reports/week-05-three-author-notes.md` | Prime setup path is documented, the lifecycle map names where Verifiers fits, and post-training terms are compared across 3 sources |
| 06 | First Prime environment | Prime-created environment folder, `project/reports/week-06-environment-spec.md`, `project/reports/week-06-three-author-notes.md` | `load_environment()` exists, 2-3 reference tasks can be scored, and environment concepts are explained across 3 sources |
| 07 | Rewards and rubrics | `project/reports/week-07-reward-checklist.md`, reward tests in the environment, `project/reports/week-07-three-author-notes.md` | 5 tasks have rubric/reward designs, reward-hacking notes, and a 3-source synthesis of reward design |
| 08 | First Prime eval | `project/runs/week-08-smoke-eval/`, `project/reports/week-08-run-schema.md`, `project/reports/week-08-three-author-notes.md` | A local Prime eval run produces scores and inspectable traces, with eval terminology reconciled across 3 sources |
| 09 | Task suite | Prime environment with 10-15 tasks, `project/tasks/week-09-coverage-table.md`, `project/reports/week-09-three-author-notes.md` | Every task has a reference answer, rubric, malformed-answer test, family label, and a 3-source task-design synthesis |
| 10 | Baseline and error analysis | `project/runs/week-10-baseline/`, `project/reports/week-10-error-analysis.md`, `project/reports/week-10-three-author-notes.md` | Baseline metrics, uncertainty, at least 5 failure examples, and a 3-author error-analysis summary are written |
| 11 | Improvement loop | `project/runs/week-11-improvement/`, `project/reports/week-11-improvement-loop.md`, `project/reports/week-11-three-author-notes.md` | One frozen-set comparison is complete, training readiness is stated, and improvement-loop concepts are triangulated |
| 12 | Final artifact | `project/reports/final-report.md`, `project/reports/week-12-three-author-notes.md`, runnable Prime eval command or demo instructions | Final report can be read without the notebook history, the environment can be rerun, and the final claims cite 3-source synthesis where useful |
